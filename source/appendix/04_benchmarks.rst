===============================
Agent 评估基准
===============================

评估是 Agent 开发的指南针。没有评估，优化就无从下手。但 Agent 的评估
比传统 NLP 任务复杂得多——不仅要测试"语言能力"，还要测试**工具调用、
多步推理、规划恢复、安全对齐**等能力。

以下是 2026 年主流的 Agent 评估基准，以及它们各自衡量什么。

SWE-bench
============

**领域**：软件工程
**发布**：2024，Princeton

SWE-bench 是目前最权威的**编码 Agent** 评估基准。它从真实 GitHub 仓库中
提取了 2294 个 Issue 和对应的 PR，要求 Agent 自动修复这些 Issue。

.. list-table::
   :header-rows: 1

   * - 维度
     - 说明
   * - 任务类型
     - 阅读 Issue 描述 → 定位代码 → 生成修复 → 通过测试
   * - 评估指标
     - 通过率（% resolved）
   * - 典型分数
     - 开源模型 10-30%，闭源模型 30-60%（2026 年数据）
   * - 难度
     - 高（需要理解整个代码库）
   * - 适合评估
     - 编码 Agent 的代码理解和修复能力

SWE-bench 的核心价值在于它测试的是**真实世界**的编程问题，而不是人造的
算法题。Agent 需要理解 Issue 描述、定位相关代码、生成修复方案、确保
测试通过——这和人类开发者的工作流程一致。

GAIA
============

**领域**：通用助理
**发布**：2023，Meta FAIR / HuggingFace

GAIA 测试 Agent 的**通用问题解决能力** ，包含 466 个需要多步推理和工具
调用的日常问题。

.. list-table::
   :header-rows: 1

   * - 维度
     - 说明
   * - 任务类型
     - 日常问题搜索与分析（如"2023 年诺贝尔奖得主中有几位女性？"）
   * - 评估指标
     - 准确率（% correct）
   * - 典型分数
     - 人类 92%，AI Agent 30-70%（2024-2026）
   * - 难度
     - 中（多步推理 + 工具调用）
   * - 适合评估
     - 通用 Agent 的综合能力

GAIA 的问题设计得很有策略：每个问题都需要至少 2-3 步推理和工具调用。
比如"找出 X 公司 2023 年营收，然后计算同比增长率"——需要搜索、分析、
计算多个步骤。如果 Agent 在某个步骤出错，整个结果都会受影响。

ToolBench
============

**领域**：工具使用
**发布**：2023，清华

ToolBench 专门测试 Agent 的**工具调用能力** ，包含超过 16,000 个需要
调用真实 API 的任务。

.. list-table::
   :header-rows: 1

   * - 维度
     - 说明
   * - 任务类型
     - 调用真实 API 完成任务（如预订酒店、查询天气）
   * - 评估指标
     - 工具选择准确率 + 参数正确率 + 任务完成率
   * - 典型分数
     - 不同模型差异大，GPT-4 约 60%
   * - 难度
     - 中高（需要理解大量 API 文档）
   * - 适合评估
     - Agent 的工具选择和执行能力

ToolBench 的独特价值在于它测试的是 Agent 在**大量工具中选择正确工具**
的能力——这和现实场景非常接近（一个 Agent 可能有几十个工具）。

AgentBench
============

**领域**：综合
**发布**：2023，清华 / 微软 / 其他

AgentBench 是一个**多维度**的 Agent 评估套件，包含 8 个不同场景
（操作系统、数据库、网络、游戏等）。

.. list-table::
   :header-rows: 1

   * - 维度
     - 说明
   * - 子任务数
     - 8 个子任务（OS、DB、Web、KG、Game 等）
   * - 评估指标
     - 各子任务的加权综合得分
   * - 典型分数
     - 开源 20-40%，GPT-4 约 60%
   * - 难度
     - 因任务而异
   * - 适合评估
     - Agent 的综合能力广度

AgentBench 的涵盖面很广，从操作系统命令到网页浏览到游戏，但它每个维度
的深度有限。适合快速评估 Agent 的"能力广度"。

BFCL（Berkeley Function Calling Leaderboard）
===============================================

**领域**：函数调用
**发布**：2024，UC Berkeley

BFCL 专注于测试模型的**函数调用**能力——这是 Agent 最核心的能力之一。

.. list-table::
   :header-rows: 1

   * - 维度
     - 说明
   * - 任务类型
     - 根据用户意图选择正确的函数和参数
   * - 评估指标
     - 函数选择准确率 + 参数匹配准确率
   * - 典型分数
     - GPT-4 约 85%，开源模型 60-80%
   * - 难度
     - 中
   * - 适合评估
     - 模型的 Function Calling 能力

HumanEval / MBPP
==================

**领域**：代码生成
**发布**：2021-2023

虽然不是专门的 Agent 基准，但它们是评估编码 Agent 基础能力的重要参考。

.. list-table::
   :header-rows: 1

   * - 基准
     - 内容
     - 指标
     - 典型分数
   * - HumanEval
     - 164 个编程问题
     - pass@k
     - GPT-4 > 90%
   * - MBPP
     - 974 个编程问题
     - pass@k
     - GPT-4 > 80%

如何选择评估基准
====================

.. list-table::
   :header-rows: 1

   * - 你的 Agent 类型
     - 推荐基准
     - 理由
   * - 编码 Agent
     - SWE-bench + HumanEval
     - 代码修复 + 代码生成
   * - 通用助理
     - GAIA + AgentBench
     - 综合推理 + 工具使用
   * - API 调用 Agent
     - ToolBench + BFCL
     - 工具选择 + 函数调用
   * - 客服 Agent
     - 自建评估集
     - 领域特定，通用基准不适合

.. admonition:: 自建评估集的重要性
   :class: tip

   通用基准很有用，但它不能替代你自己任务的评估。建议：
   1. 用通用基准对模型做**初筛** （选模型阶段）
   2. 自建 50-100 个业务场景的评估集做**终测** （上生产前）
   3. 持续监控生产环境中的**任务成功率** （上线后）

   一个自建评估集的价值远高于任何通用基准——因为它衡量的是
   "你的 Agent 能不能完成你的任务"。
